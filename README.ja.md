Macをロックした後や、ディスプレイを閉じた後も長時間の処理を継続します。

FreeThumbは、ローカルAIエージェント、ビルド、ダウンロード、レンダリングなど、
スリープで中断されたくない処理のための軽量なmacOSメニューバーアプリです。

## 主な機能

- 30分、1時間、2時間、4時間、または手動で停止するまでMacをスリープさせません
- 画面ロック中やディスプレイを閉じた状態でも処理を継続します
- 停止時に以前のスリープ設定を復元します
- バッテリー、電源、低電力モード、ディスプレイ、熱状態を表示します
- システム負荷と、バッテリー温度／総電力の2本線グラフを任意で表示できます
- ローカル通知、iMessage、Mail、HTTPS Webhookに対応します
- バッテリー消費、熱状態、フォアグラウンドアプリとの概算関係を記録できます
- 10言語とログイン時の自動起動に対応します

## 動作環境

- macOS 13以降
- 閉じた状態での保護を初めて有効にする際の管理者承認

ディスプレイ制御にmacOSのプライベートフレームワークを使用するため、Mac App
Storeでは配布できません。

## ダウンロード

[GitHub Releases](../../releases/latest)を開き、使用するプラットフォーム向けの
パッケージをダウンロードしてください。利用可能なmacOS版とWindows版が表示されます。

macOSではDMGを開き、`FreeThumb.app`をApplicationsフォルダへドラッグします。
FreeThumbを検証できないという警告が出た場合は、ゴミ箱へ移動しないでください。
一度起動を試した後、**システム設定 → プライバシーとセキュリティ**の「セキュリティ」
で**このまま開く**を選び、**開く**を確認します。信頼できる本リポジトリから取得した
パッケージに限って実行してください。

## ソースからビルド

必要に応じてXcode Command Line Toolsをインストールします。

```sh
xcode-select --install
```

`apps/macos` ディレクトリでビルドして起動します。

```sh
cd apps/macos
./scripts/build-app.sh
open dist/FreeThumb.app
```

ドラッグインストール用DMGを作成する場合：

```sh
cd apps/macos
./scripts/build-dmg.sh
```

FreeThumbはメニューバーに表示され、Dockには表示されません。

## クイックスタート

1. メニューバーの親指アイコンをクリックします。
2. 時間を選択します。`∞`は手動で停止するまで有効です。
3. **スリープ防止モードを有効にする**をクリックします。
4. 初回のみmacOSの管理者ダイアログを承認します。
5. `Control-Command-Q`でロックするか、ディスプレイを閉じます。処理は継続します。
6. 処理が終わったらFreeThumbを開き、**スリープ防止モードを無効にする**をクリックします。

処理が電源やネットワークを必要とする場合は、接続を維持してください。FreeThumbは
スリープを防ぎますが、停電、通信切断、アプリのクラッシュ、強制終了は防げません。

## アイコンの色

| 色 | 状態 |

| グレー | 保護は停止中 |
| 緑 | 保護は正常に動作中 |

## 設定と高度な機能

FreeThumbメニュー下部の歯車ボタンをクリックします。

### 一般

- **言語：**システム設定に従うか、英語、中国語、日本語、韓国語などを選べます。
- **ログイン時に起動：**macOSへのログイン後に自動起動します。
- **メニューバー監視：**システム負荷、バッテリー温度／総電力、高負荷アプリの表示を
  個別に切り替えます。
- **アップデート確認：**プロジェクトが提供するHTTPSマニフェストURLを入力して
  手動確認します。自動インストールは行いません。

グラフは30秒ごとにサンプリングし、最大24時間分をメモリに保持します。バッテリー
グラフは左軸が温度、右軸が総電力です。バッテリー使用時はバッテリー出力、AC接続時は
充電電力を含むアダプタ総入力を使用します。高電力リストには現在のユーザーが実行する
サードパーティ製アプリのみを
表示し、macOSの`top`サンプルの実行中平均をアプリ単位で集計して順位付けします。
数値は相対推定値であり、ワットではありません。このウィジェットが有効な場合のみ、
バックグラウンドで5分ごと、メニュー表示中は1分ごとにサンプリングします。

### 保護

バッテリー警告／緊急しきい値と、次の通知条件を設定できます。

- AC電源の切断
- 低電力モードの有効化
- 深刻または危機的な熱状態の継続
- セッション終了前

これらは通知のみを行い、保護を停止しません。変更は次回の開始時に反映されます。

### 通知

ローカル通知は初期状態で有効です。iMessage、Mail、HTTPS Webhookも選択できます。
iMessageとメールには各macOSアプリで設定済みのアカウントが必要です。

使用前に**ローカル通知をテスト**または**有効なチャンネルをテスト**を実行して
ください。送信エラーが発生しても保護中の処理は中断されません。

### アクティビティ

**アクティビティ統計を記録**を有効にすると、保護中に1分ごとに次を記録します。

- セッション時間とバッテリー消費率
- バッテリー残量の推移
- macOSの熱状態の推移
- フォアグラウンドアプリとの概算関係

各区間は開始時に前面にあったアプリへ割り当てられます。アプリの実際の消費電力を
測定する機能ではありません。不要な場合は記録を無効にできます。

## 管理者承認が必要な理由

macOSの閉じた状態でのスリープ設定を変更するには管理者権限が必要です。初回のみ、
FreeThumbは`/private/etc/sudoers.d/freethumb`に限定的なルールを追加し、現在の
ユーザーに次の2コマンドだけを許可します。

```text
pmset disablesleep 0
pmset disablesleep 1
```

FreeThumbはパスワードを読み取ったり保存したりできません。設定変更後に異常終了した
場合は、ウォッチドッグが通常のスリープ設定を復元します。

## プライバシーと負荷

- 監視データとアクティビティデータはMacのメモリ内にだけ保持されます
- 明示的に有効にした通知チャンネルだけが外部へデータを送信します
- 高負荷アプリ推定は初期状態で無効です
- プロセス走査は機能が有効でメニューが表示されている間だけ実行されます
- 通常の監視には上限付きの低頻度サンプリングを使用します

## トラブルシューティング

**ローカル通知が表示されない：** **設定 → 通知**で状態を更新し、必要に応じて
macOSの**システム設定 → 通知**でFreeThumbを許可してください。

**AC接続時に総電力が高い：** 正常な動作です。AC値にはシステム負荷とバッテリーへ
流れる充電電力の両方が含まれます。

**保護を開始できない：** メニューのエラーを確認し、管理者ダイアログを承認したこと
と、現在のアカウントが管理者であることを確認してください。

**スリープ設定が復元されない：** メニューから停止した後もエラーが続く場合は、
ターミナルで次を実行します。

```sh
sudo pmset disablesleep 0
```
