让 Mac 在锁屏或合盖后继续执行长时间任务。

FreeThumb 是一款轻量级 macOS 菜单栏应用，适合本地 AI Agent、编译、下载、
渲染等不能被系统睡眠中断的任务。它还会监控电池和热状态，但不会擅自停止任务。

## 主要功能

- 可保持唤醒 30 分钟、1 小时、2 小时、4 小时或无限时间
- 锁屏和合盖后继续运行任务，并在停止保护时恢复原来的睡眠设置
- 显示电池、电源、低电量模式、上盖和热压力状态
- 可选系统压力图，以及共用一个绘图区的电池温度/总功率双曲线图
- 支持本地通知、iMessage、邮件和 HTTPS Webhook 告警
- 可记录电池消耗、热压力和前台应用的近似关联
- 支持十种界面语言和登录时启动

## 系统要求

- macOS 13 或更高版本
- 首次启用合盖保护时需要管理员批准

由于合盖显示控制使用了 macOS 私有框架，FreeThumb 不通过 Mac App Store 分发。

## 下载

打开 [GitHub Releases](../../releases/latest)，下载对应平台的软件包。可用的 macOS
和 Windows 软件包都会列在这里。

macOS 用户打开 DMG，将 `FreeThumb.app` 拖到 Applications 文件夹。如果系统提示无法
验证 FreeThumb，请不要移到废纸篓；尝试打开后前往**系统设置 → 隐私与安全性**，在
“安全性”中选择**仍要打开**，再确认**打开**。仅对从本仓库下载且确认可信的软件包
执行此操作。希望自行构建的用户可继续阅读下一节。

## 从源码构建

如未安装 Xcode Command Line Tools，请先运行：

```sh
xcode-select --install
```

在 `apps/macos` 目录中构建并打开应用：

```sh
cd apps/macos
./scripts/build-app.sh
open dist/FreeThumb.app
```

如需生成拖拽安装的 DMG：

```sh
cd apps/macos
./scripts/build-dmg.sh
```

FreeThumb 会显示在菜单栏中，不会出现在 Dock 中。

## 快速使用

1. 点击菜单栏中的拇指图标。
2. 选择保护时长；选择 `∞` 可一直运行到手动停止。
3. 点击**开启防睡眠模式**。
4. 首次使用时批准 macOS 管理员提示。
5. 按 `Control-Command-Q` 锁屏，或直接合盖，任务会继续运行。
6. 任务结束后打开 FreeThumb，点击**关闭防睡眠模式**。

如果任务依赖电源或网络，请确保它们保持连接。FreeThumb 只能阻止系统睡眠，无法避免
断电、断网、应用崩溃或强制关机。

## 设置与进阶功能

点击 FreeThumb 菜单底部的齿轮按钮。

### 通用

- **语言：**跟随系统，或手动选择英语、简体中文、日语、韩语等语言，立即生效。
- **登录时启动：**登录 macOS 后自动运行 FreeThumb。
- **菜单栏监控：**控制系统压力图、电池温度/总功率图和高能耗应用估算是否显示。
- **检查更新：**填写项目提供的 HTTPS 发布清单地址后手动检查；不会静默安装。

图表每 30 秒采样一次，在内存中最多保留 24 小时。电池组合图左轴为温度，右轴为
总功率。使用电池时读取电池输出；接通 AC 时读取适配器总输入，其中也包含电池充电
功率。高能耗列表只包含当前用户
运行的第三方应用，并按 macOS `top` 采样值的运行期平均数排序；同一应用的多个进程会
合并计算。该数字是相对估值，不是瓦特。仅在组件开启时采样：后台每 5 分钟一次，菜单
打开时每分钟一次。

### 保护

可设置电池警告和紧急阈值，并选择以下告警条件：

- 电源适配器断开
- 低电量模式开启
- 严重或危急热压力持续一段时间
- 会话即将结束

这些条件只会发出告警，不会停止保护。修改会在下次启动保护时生效。

### 告警

本地通知默认开启。还可以选择 iMessage、Mail 邮件或 HTTPS Webhook。iMessage 和
邮件需要系统应用中已有可用账户，macOS 可能请求自动化权限。

正式使用前请点击**测试本地通知**或**测试已启用渠道**。投递失败会显示在
FreeThumb 中，但不会中断受保护的任务。

### 活动记录

开启**记录活动统计**后，保护期间每分钟记录一次：

- 会话时长和电池消耗百分点
- 电池电量曲线
- macOS 热压力曲线
- 前台应用近似关联

每个时间区间只会归入区间开始时的前台应用，这不代表该应用的真实能耗。不需要这些
数据时，可以关闭活动记录。

## 为什么需要管理员批准

macOS 要求管理员权限才能修改合盖睡眠设置。首次使用时，FreeThumb 会安装一个范围
严格受限的规则 `/private/etc/sudoers.d/freethumb`，只允许当前用户执行：

```text
pmset disablesleep 0
pmset disablesleep 1
```

FreeThumb 无法读取或保存你的密码。如果应用在修改设置后意外退出，看门狗会恢复正常
睡眠设置。

## 隐私与能耗

- 监控和活动数据只保留在本机内存中
- 只有你主动启用远程告警渠道时才会发送数据
- 高能耗应用估算默认关闭，并且只在启用且菜单可见时扫描
- 常规监控使用有上限的低频采样

## 常见问题

**本地通知没有出现：**打开**设置 → 告警**并刷新状态；如被阻止，请在
**系统设置 → 通知**中启用 FreeThumb。

**接通 AC 后总功率更高：**这是正常现象；AC 数值同时包含当前系统负载和流入电池的
充电功率。

**保护无法启动：**查看菜单中的错误信息，确认管理员提示已批准且当前账户具有管理员
权限。

**睡眠设置没有恢复：**先尝试从菜单停止保护。如仍提示恢复失败，在终端运行：

```sh
sudo pmset disablesleep 0
```
