Mac을 잠그거나 덮개를 닫은 뒤에도 장시간 작업을 계속 실행합니다.

FreeThumb은 로컬 AI 에이전트, 빌드, 다운로드, 렌더링처럼 시스템 잠자기로 중단되면
안 되는 작업을 위한 가벼운 macOS 메뉴 막대 앱입니다.

## 주요 기능

- 30분, 1시간, 2시간 또는 직접 중지할 때까지 Mac을 깨운 상태로 유지
- 화면 잠금 및 덮개가 닫힌 상태에서도 작업 계속 실행
- 보호 종료 시 이전 잠자기 설정 복원
- 배터리, 전원, 저전력 모드, 덮개 및 열 상태 표시
- 시스템 압력과 배터리 온도/전력의 독립된 두 선을 하나의 차트에 표시
- 로컬 알림, iMessage, Mail 및 HTTPS 웹훅 지원
- 배터리 사용량, 열 압력 및 전면 앱의 대략적인 연관 관계 기록
- 10개 언어와 로그인 시 자동 실행 지원

## 요구 사항

- macOS 13 이상
- 덮개 닫힘 보호를 처음 활성화할 때 관리자 승인

덮개 디스플레이 제어에 macOS 비공개 프레임워크를 사용하므로 Mac App Store를 통해
배포할 수 없습니다.

## 다운로드

[GitHub Releases](../../releases/latest)를 열고 사용할 플랫폼의 패키지를
다운로드하십시오. 제공되는 macOS 및 Windows 패키지가 여기에 표시됩니다.

macOS에서는 압축을 푼 뒤 `FreeThumb.app`을 엽니다. 직접 빌드하려면 다음 절차를
따르십시오.

## 소스에서 빌드

필요한 경우 Xcode Command Line Tools를 설치합니다.

```sh
xcode-select --install
```

프로젝트 디렉터리에서 앱을 빌드하고 실행합니다.

```sh
./scripts/build-app.sh
open dist/FreeThumb.app
```

FreeThumb은 메뉴 막대에 표시되며 Dock에는 표시되지 않습니다.

## 빠른 시작

1. 메뉴 막대에서 엄지 아이콘을 클릭합니다.
2. 시간을 선택합니다. `∞`를 선택하면 직접 중지할 때까지 유지됩니다.
3. **보호 시작**을 클릭합니다.
4. 처음 사용할 때 macOS 관리자 요청을 승인합니다.
5. `Control-Command-Q`로 Mac을 잠그거나 덮개를 닫습니다. 작업은 계속 실행됩니다.
6. 작업이 끝나면 FreeThumb을 열고 **보호 중지**를 클릭합니다.

작업에 전원이나 네트워크가 필요하다면 연결을 유지하십시오. FreeThumb은 잠자기를
방지하지만 정전, 네트워크 끊김, 앱 충돌 또는 강제 종료는 방지할 수 없습니다.

## 아이콘 색상

| 색상 | 의미 |
| --- | --- |
| 회색 | 보호 꺼짐 |
| 녹색 | 보호가 정상적으로 실행 중 |
| 노란색 | 확인이 필요한 경고 있음 |
| 빨간색 | 심각한 상태 또는 복원 오류 |

## 설정 및 고급 기능

FreeThumb 메뉴 아래쪽의 톱니바퀴 버튼을 클릭합니다.

### 일반

- **언어:** 시스템 설정을 따르거나 영어, 중국어, 일본어, 한국어 등을 선택합니다.
- **로그인 시 실행:** macOS 로그인 후 FreeThumb을 자동 실행합니다.
- **메뉴 막대 모니터링:** 시스템 압력, 배터리 온도/전력 및 고에너지 앱 추정 표시를
  선택합니다.
- **업데이트 확인:** 프로젝트에서 제공하는 HTTPS 릴리스 매니페스트 URL을 입력해
  수동으로 확인합니다. 업데이트를 자동 설치하지 않습니다.

차트는 1분마다 샘플링하고 메모리에 최대 24시간의 데이터를 보관합니다. 배터리 차트의
왼쪽 축은 온도, 오른쪽 축은 전력입니다. 전원 어댑터 연결 중에는 배터리 방전 전력을
사용할 수 없을 수 있습니다. 고에너지 목록에는 현재 사용자가 실행한 타사 앱만 표시되며
macOS의 프로세스별 에너지 카운터로 정렬됩니다. 활성 상태 보기의 비공개 에너지 영향
점수를 재현한다고 주장하지 않습니다.

### 보호

배터리 경고 및 긴급 임계값과 다음 알림 조건을 설정할 수 있습니다.

- 전원 어댑터 연결 해제
- 저전력 모드 활성화
- 심각하거나 위급한 열 압력 지속
- 세션 만료 임박

이 조건은 알림만 발생시키며 보호를 중지하지 않습니다. 변경 사항은 다음 보호 시작 시
적용됩니다.

### 알림

로컬 알림은 기본으로 활성화됩니다. iMessage, Mail 또는 HTTPS 웹훅도 선택할 수
있습니다. iMessage와 이메일은 해당 macOS 앱에 설정된 계정이 필요합니다.

사용하기 전에 **로컬 알림 테스트** 또는 **활성화된 채널 테스트**를 실행하십시오.
전송 오류가 발생해도 보호 중인 작업은 중단되지 않습니다.

### 활동

**활동 통계 기록**을 활성화하면 보호 중 1분마다 다음 항목을 기록합니다.

- 세션 시간 및 배터리 사용 퍼센트포인트
- 배터리 잔량 변화
- macOS 열 압력 변화
- 전면 앱과의 대략적인 연관 관계

각 구간은 시작할 때 전면에 있던 앱에 할당됩니다. 앱의 실제 에너지 사용량을 측정하는
기능은 아닙니다. 필요하지 않으면 기록을 끌 수 있습니다.

## 관리자 승인이 필요한 이유

macOS의 덮개 닫힘 잠자기 설정을 변경하려면 관리자 권한이 필요합니다. 처음 사용할 때
FreeThumb은 `/private/etc/sudoers.d/freethumb`에 범위가 제한된 규칙을 설치하며,
현재 사용자에게 다음 두 명령만 허용합니다.

```text
pmset disablesleep 0
pmset disablesleep 1
```

FreeThumb은 암호를 읽거나 저장할 수 없습니다. 설정을 변경한 뒤 앱이 예기치 않게
종료되면 감시 프로세스가 정상 잠자기 설정을 복원합니다.

## 개인정보 보호 및 자원 사용

- 모니터링 및 활동 데이터는 Mac의 메모리에만 보관됩니다
- 직접 활성화한 원격 알림 채널만 외부로 데이터를 전송합니다
- 고에너지 앱 추정은 기본적으로 비활성화됩니다
- 프로세스 검색은 기능이 활성화되고 메뉴가 보일 때만 실행됩니다
- 일반 모니터링에는 제한된 저빈도 샘플링을 사용합니다

## 문제 해결

**로컬 알림이 표시되지 않음:** **설정 → 알림**에서 상태를 새로 고친 뒤 필요한 경우
macOS **시스템 설정 → 알림**에서 FreeThumb을 허용합니다.

**배터리 전력을 사용할 수 없음:** 전원 어댑터가 연결되어 있으면 일반적으로 배터리 쪽
방전 전력을 사용할 수 없습니다.

**보호가 시작되지 않음:** 메뉴의 오류를 확인하고 관리자 요청을 승인했는지, 현재 계정에
관리자 권한이 있는지 확인합니다.

**잠자기 설정이 복원되지 않음:** 메뉴에서 보호를 중지한 뒤에도 오류가 계속되면
터미널에서 다음을 실행합니다.

```sh
sudo pmset disablesleep 0
```
